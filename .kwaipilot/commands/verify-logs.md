# Debug 日志验证

自动验证代码中的 debug 日志是否符合 debug-logging-standard 规范，并生成详细的验证报告。

## 触发关键词

当用户说以下内容时，应使用 verify-debug-logs skill：
- "验证日志"
- "检查日志规范"
- "生成日志报告"
- "日志质量检查"
- "审查日志"
- "评估日志"

## 使用方法

```
/verify-logs [文件路径]
```

如果不指定文件路径，将验证当前打开的文件。

## 报告内容

验证报告包含：
- 📊 总体评分（0-100分）
- ✅ 通过的检查项
- ❌ 未通过的检查项
- ⚠️ 警告和建议
- 💡 改进建议（按优先级排序）
- 🎯 下一步行动清单

## 示例

**验证单个文件**:
```
/verify-logs HabitKit/Models/HabitStatistics.swift
```

**验证当前文件**:
```
验证当前文件的日志规范
```

**验证整个目录**:
```
验证 HabitKit/Models 目录下所有文件的日志
```

## 评分标准

- **90-100 分**: ✅ 优秀
- **75-89 分**: ✅ 良好
- **60-74 分**: ⚠️ 及格
- **60 分以下**: ❌ 不及格
