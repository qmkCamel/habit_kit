# 测试验证

在手动测试或自动化测试完成后，通过分析测试日志和代码，自动验证功能是否正常执行，并生成详细的验证报告。

## 触发关键词

当用户说以下内容时，应使用 test-verification skill：
- "验证测试结果"
- "检查功能是否正常"
- "分析测试日志"
- "测试是否通过"
- "功能验证"
- "测试报告"

## 使用方法

```
/verify-test [功能名称]
```

然后AI会询问测试日志和相关信息。

## 需要提供的信息

1. **功能名称**: 测试的功能（如：统计功能）
2. **测试场景**: 具体的测试步骤（如：点击统计按钮查看数据）
3. **测试日志**: 控制台输出或日志文件
4. **相关代码**: 相关的源文件（可选，AI会自动识别）

## 验证内容

报告包含：
- 📊 验证总览（流程、顺序、数据、异常）
- ✅ 通过的检查项
- ❌ 发现的问题
- ⚠️ 警告和建议
- 📈 性能分析
- 🎯 改进建议
- 💯 总体评分和结论

## 示例

**手动测试验证**:
```
用户: 我测试了统计功能，这是日志：
[粘贴日志]

AI: （分析日志和代码）生成验证报告
```

**自动化测试验证**:
```
用户: /verify-test 统计功能

AI: 请提供测试日志

用户: [粘贴 XCTest 输出]

AI: （验证并生成报告）
```

## 评分标准

- **90-100 分**: ✅ 优秀
- **75-89 分**: ✅ 良好
- **60-74 分**: ⚠️ 及格
- **60 分以下**: ❌ 不及格

## 前置要求

为获得最佳验证效果：
- ✅ 代码中有符合 debug-logging-standard 的日志
- ✅ 测试生成了完整的日志输出
- ✅ 日志记录了关键数据和执行路径
